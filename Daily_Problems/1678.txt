/*
ğŸ’¡ Problem: Goal Parser Interpretation
-------------------------------------
You are given a string command that represents instructions from a "Goal Parser".

The command string consists of the following patterns:
1. "G" â†’ represents the string "G"
2. "()" â†’ represents the string "o"
3. "(al)" â†’ represents the string "al"

You need to interpret the command and return the final interpreted string.

ğŸ§© Example:
Input: "G()(al)"
Output: "Goal"

Explanation:
- "G" â†’ "G"
- "()" â†’ "o"
- "(al)" â†’ "al"
â†’ Final output = "Goal"
*/


class Solution {
    public String interpret(String command) {

        // âœ… Using StringBuilder because it is efficient for string concatenation
        StringBuilder ans = new StringBuilder();

        // ğŸ” Loop through each character in the command string
        for (int i = 0; i < command.length(); i++) {

            // Case 1: If the current character is 'G', directly append 'G'
            if (command.charAt(i) == 'G') {
                ans.append('G');
            }

            // Case 2: If we find '(', check what comes next
            if (command.charAt(i) == '(') {

                // If next character is ')', that means the pattern is "()"
                // So append 'o' to the answer
                if (command.charAt(i + 1) == ')') {
                    ans.append('o');
                    // Skip the next character ')' since it's already processed
                    i++;
                }

                // Otherwise, the pattern is "(al)"
                // So append "al" and skip the next 3 characters "al)"
                else {
                    ans.append("al");
                    i += 3;
                }
            }
        }

        // Convert the StringBuilder to a normal string and return it
        return ans.toString();
    }
}

/*
ğŸ§  Approach Summary:
--------------------
1ï¸âƒ£ Use a StringBuilder to build the result efficiently.
2ï¸âƒ£ Loop through the string character by character.
3ï¸âƒ£ Check each pattern:
    - 'G' â†’ add 'G'
    - '()' â†’ add 'o'
    - '(al)' â†’ add 'al'
4ï¸âƒ£ Adjust index `i` to skip over already-processed parts.

â±ï¸ Time Complexity: O(n)
   (We visit each character once)
ğŸ’¾ Space Complexity: O(n)
   (StringBuilder stores the final result)

âœ… Clean, simple, and efficient!
